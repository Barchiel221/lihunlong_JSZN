# 空中具身智能挑战赛(创意组)国赛计划书

> 机型:220mm 轴距四旋翼(机身最宽含桨约 30cm)
> 软件基线:lihunlong_ws2 现有栈(MID360 + FAST-LIO2 + EGO-Planner + PX4)
> 依据:《2026 第九届中国高校智能机器人创意大赛·空中具身智能赛项规则》修订版
> 日期:2026-06-12

## 一、规则要点速览(只列影响方案的条目)

| 项目 | 规则 | 对我们的含义 |
|---|---|---|
| 场地 | 8m×8m×**2m**,四周防护网 | 飞行高度窗口窄(顶 2m 是网,碰网也扣分);地图范围可收小 |
| 区域① | 8–10 个 0.5×0.5×2m 立柱,随机分布 | 与省赛同类,现有栈已验证 |
| 区域② | 5–7 棵 ≥1.8m 假树 + 角落风扇风扰 | 非结构化点云 + 抗风,树高 1.8m 接近 2m 顶,基本只能绕不能翻 |
| 区域③ | 2 个 0.5×0.5×2m 移动障碍,≤0.5m/s | EGO v1 无动态预测,靠高频重规划 + 残影清除 |
| 区域④ | 拱形障碍,可通过开口宽 **0.7m** | 最窄瓶颈,直接约束膨胀参数(详见 §4.2) |
| 得分环 | 每区域后 1 个,半径 0.5m,**中心位置赛前给出** | 航点序列赛前填参,环口直径 1m,中心对准误差预算 ±0.3m |
| 起飞/降落区 | 0.5×0.5m;起飞区为原点 [0,0],降落区赛前给出 | 精确降落:停桨后 4 电机投影全在 KT 板内得 10 分 |
| 计分 | 起飞 10 + 4 环 ×18 + 降落 10 + 碰撞分 8(每碰 -2),满分 100 | **完整飞完是碰撞分 8 分的前提**;两轮取最高 |
| 计时 | 限时 5 分钟;同分比最后得分时刻 | 时间只在同分时起作用 → 第一优先级是稳,不是快 |
| 设备 | 四旋翼,150≤轴距≤330mm,≤6S,续航≥5min,**全程自主不得遥控**,必须一键停桨 | 220mm 合规;RC 只保留急停(kill);触地前接管遥控 = 0 分 |
| 现场 | 上交自备路由器,统一比赛路由器;10min 设备调试;5min 备赛 | goal 不能依赖人在 RViz 上点 → 航点必须机载自动下发 |

## 二、任务流程与得分策略

```
起飞区[0,0] 起飞(10) → 区域①立柱阵 → 环①(18)
→ 区域②树林+风扇 → 环②(18)
→ 区域③移动障碍 → 环③(18)
→ 区域④0.7m拱门 → 环④(18)
→ 降落区精确降落(10) → 自动停桨,计时结束
```

策略原则:

1. **稳 > 快**。排名先比总分,时间只在同分时比。基线速度沿用实飞验证过的 max_vel≈0.6,跑通 100 分后再提速。
2. **完整飞行优先**。碰撞分 8 分要求"完整飞行",中途坠落则碰撞分清零且降落 10 分归零——任何冒险动作的期望收益都是负的。
3. **两轮差异化**:第一轮"保守档"确保拿到完整成绩;第一轮已得高分(≥90)时第二轮才切"提速档"冲时间/补环。
4. 环中心、降落区位置赛前公布 → 做成 **YAML 任务文件**,现场 10 分钟调试时间内只改一个文件,不动代码。

## 三、平台改型(330mm → 220mm)

### 3.1 几何与质量

| 项 | 旧机(已实飞) | 新机(目标) |
|---|---|---|
| 轴距 | 330mm | 220mm |
| 含桨最大外廓 | 50×40cm,R≈0.25m | 最宽约 30cm,**R≈0.15m** |
| 电机离机心投影半径 | ~165mm | ~110mm |

### 3.2 硬件清单与约束

- 机载计算:Orange Pi 5(沿用);LiDAR:MID360(~265g,含线)。220mm 机架挂 MID360 + OPi5 总重显著占比 → **桨/电机选型必须留 ≥2:1 推重比**,否则区域②风扰下姿态余量不足。
- 电池 ≤6S,**实测续航 ≥5min**(规则硬性要求,检录会查);按比赛 5 分钟限时 + 备赛悬停,建议实际续航 ≥8min。
- MID360 安装:尽量保持与旧机相同的安装姿态(水平、机心正上/正前),FAST-LIO 外参(`mid360.yaml` 里的 extrinsic)按新机实测重标。
- 一键停桨:RC 保留且**只绑 PX4 kill switch**(规则强制要求;除失控外触地前碰摇杆 = 0 分)。赛前录制停桨功能演示流程,检录要看。
- 振动:220mm 机架刚性高但桨距小转速高,IMU 减振垫按需加;FAST-LIO 对 IMU 噪声敏感,改机后先静态/手持采 bag 验证里程计漂移再上飞。

### 3.3 改型验证步骤(上飞前)

1. 台架:称重、推力曲线、续航放电测试(记录,检录材料)。
2. 手持/推车采 bag:验证 FAST-LIO 外参与时间同步,漂移 <1% 行程。
3. 悬停试飞:EKF2 吃 `/fmu/in/vehicle_visual_odometry` 的位置保持精度,对比旧机基线。
4. 一键停桨地面测试 + 低空测试。

## 四、软件栈适配(基于 ego_real_flight.launch.py 现参数)

### 4.1 体积相关参数重整(核心变更)

旧参数按 R≈0.25m 标定,新机 R≈0.15m,整体可收 0.1m:

| 参数 | 现值(330mm) | 建议初值(220mm) | 依据 |
|---|---|---|---|
| `obstacles_inflation` | 0.28 | **0.20** | R0.15 + voxel 量化 0.05(resolution=0.1) |
| `obstacles_inflation_z` | 0.20 | **0.15** | 机高更矮 |
| `dist0` | 0.08 | **0.08(不变)** | 软余量与体积无关,继续兜优化抖动 |
| `precision_xy_tolerance` | 0.12 | **0.10** | 电机投影半径 0.11m,KT 板 0.5m → 允许中心误差 0.25-0.11=0.14m,取 0.10 留余量保 4 电机全进板(10 分 vs 8 分) |

**0.7m 拱门通过性验算**:0.7 − 2×0.20(inflation) = 0.30m 自由走廊;dist0=0.08 软推后机心可用 ±0.07m;机体半宽 0.15 < 0.35−0.20 ✓。旧机(inflation 0.28)走廊只剩 0.14m 基本不可行——**这就是改 220mm 的核心收益,拱门区从"赌"变成"常规航段"**。

### 4.2 高度走廊参数(待环中心公布后定)

场地顶 2m 是防护网,树高 ≥1.8m:

- `virtual_ceil_height`:现 1.5。若有环中心 >1.3m 需上调;上限建议 **1.7**(实际峰值 ≈ ceil−0.23,离网保 0.5m+)。
- `virtual_ground_height` / `enable_height`:现 0.45/0.55。**侧视图显示环④可能很低**(拱门腿高约 0.5m 量级),若公布的环中心 <0.7m,该航段需把 virtual_ground 降到 0.25/0.35 或做分段参数。
- 实现方式:把 ceil/ground 做成**按航段切换的参数**(任务文件每个 waypoint 附带高度走廊),而不是全程一套——这是赛前唯一需要写的新功能,工作量小、收益直接。

### 4.3 任务调度(替代 RViz 手动发 goal)

规则要求全程自主 + 路由器受控,不能依赖地面站点 RViz:

- 复用 `top_launch_pkg` 已有的 `ego_multi_goal.launch.py` / `multi_waypoint_publisher.py`,扩成读 **mission.yaml**(4 个环中心 + 降落区坐标,赛前现场填写)依次发 `/move_base_simple/goal`。
- 到点判定沿用 `px4_offboard_pkg` 现有 `ego_goal_reach_*` 容差;**环只在中间航点用粗容差(xy 0.35)直接穿过不悬停**,最后降落区才走 FINAL_CORRECTION(0.10/0.15)+ 自动降落停桨链路(已实飞验证)。
- 穿环姿态:goal 设在环中心,前后各加一个引导航点(环面法向 ±0.8m)保证垂直穿环,避免擦框(擦环框=碰撞,-2)。
- 一键启动:裁判示意后单条命令拉起整套(`enable_arm:=true`),起飞自动开始。

### 4.4 分区域策略

| 区域 | 策略 | 参数动作 |
|---|---|---|
| ① 立柱阵 | 现有栈强项,按省赛经验直接飞 | 基线参数 |
| ② 树林+风扇 | 树冠点云稀碎 → inflation 已兜;风扰交给 PX4 内环,**本段限速**(航段级 max_vel 0.5)减小轨迹跟踪误差;远离风扇角落一侧规划(lambda 不改,航点布在背风侧) | 航段限速 |
| ③ 移动障碍 | EGO v1 无预测,对策:a) `local_update_range` 收到 6m 减少残影;b) grid_map raycast 清空已验证可清除移动障碍尾迹;c) 路线穿越方向与障碍运动路径垂直、快速通过;d) 0.5m/s 很慢,replan 频率足够 | 收 local_update_range;必要时提高 replan 触发灵敏度 |
| ④ 0.7m 拱门 | §4.1 验算已通;拱门前设引导航点对正开口中心,**低速直线穿越**(航段 max_vel 0.4),禁止在洞口内重规划转弯 | 航段限速 + 引导航点 |

### 4.5 不动的东西

- FAST-LIO / lio_px4_bridge / ego_px4_adapter / offboard 状态机主体:已实飞验证,不重构(§4.6 仅改 FLY 转发段的 yaw 赋值,不动状态机转换/精降链路)。
- `use_distinctive_trajs` 保持 False(SIGSEGV bug)。
- 调参流程沿用 bag 复盘工作流(`bag_analysis/`),**每次试飞全量录 bag**,改参数前先复盘——这同时也是检录/申诉的证据材料。
- `ego_replay.launch.py` 与 `ego_real_flight.launch.py` 参数继续双向同步。

### 4.6 机头朝向(yaw)策略

**问题**:省赛为压掉"起飞后机头猛甩约 90°",把 offboard FLY 转发段的 yaw 硬锁成起飞朝向 `home_yaw_`,全程平移不转头。国赛地图区域②→③需右转 90°,锁死后**区域③④将倒着飞**,不可接受。

**根因**:EGO traj_server 期望 yaw 取路径切向,而其内部 `last_yaw_` 初值为 0(非飞机真实起飞朝向),起飞第一条轨迹一生成 yaw 即从 0 跳向切向、PX4 猛甩;省赛的锁死是把这条 yaw 完全挡在外面。

**决策**:改为**速度死区 + 转向限速的机头跟随**,在 offboard 转发段自算 yaw(不动 traj_server 与 ENU↔NED 两处),从飞机真实起飞朝向播种:

- 起飞/悬停(速度 <0.3m/s)→ 机头保持,**不 snap**;
- 巡航 → 机头以 ≤60°/s 平滑跟随飞行方向,区域②→③右转自然转过去而非倒飞;
- 穿环/拱门 → 沿引导航点法向直线运动,机头自然正对开口垂直穿越。

**安全性**:得分环节不含朝向(起飞 10+4 环×18+降落 10+碰撞 8),本改动纯为消除倒飞;且 MID360 360° 水平视野下机头朝向不影响避障,故 yaw 跟随只需"看起来对且稳"、不承担避障功能。参数(`yaw_follow_speed`/`yaw_rate_max_deg`)与代码落点见《国赛代码改动方案》改动 I;`yaw_follow_speed` 设极大即退化为锁死 `home_yaw_`(省赛行为)作为兜底。必须先 dry-run 录 bag 逐段验证(起飞无阶跃、转角单调)再上真飞。

## 五、现场执行预案

1. **赛前准备包**:mission.yaml 模板(环坐标留空)、三套参数档(保守/标准/提速,launch argument 切换)、一键启动脚本、停桨演示流程卡。
2. **报到日自由练习 + 两次 10min 调试**:第一次调试只做两件事——FAST-LIO 在真实场地建图质量确认、按公布的环中心填 mission.yaml 后跑一遍 dry-run(`enable_arm:=false`,看 RViz 轨迹);第二次调试留给修问题。
3. **检录**:轴距/电池/停桨演示;设备核验后不得更换 → 备机也要在核验前定下来(规则只封存参赛设备,备件策略赛前问裁判)。
4. **第一轮(保守档)**:max_vel 0.6,目标完整 100 分。
5. **第二轮决策树**:第一轮 ≥90 → 提速档(max_vel 0.8–1.0,仅①③区段提速,②④不提);第一轮丢环/出事 → 保持保守档修掉具体问题再跑。
6. **急停纪律**:操作员全程手握 RC 只待命 kill;裁判判失控/坠机立即停桨(不停桨 = 0 分)。

## 六、风险清单

| 风险 | 影响 | 缓解 |
|---|---|---|
| 220mm 挂 MID360 推重比不足 | 风扰区姿态发散 | 台架推力实测,不达 2:1 换动力套 |
| 改机后 FAST-LIO 外参/振动问题 | 里程计漂移→全任务失败 | §3.3 手持 bag 验证先行,IMU 减振 |
| 防护网被 LiDAR 稀疏感知 | 贴边航段误入网区(-2/次) | 地图边界硬约束:map_size 收到场地实际尺寸 + 航点离边 ≥1m |
| 移动障碍残影堵路 | 规划停滞/绕远 | local_update_range 收小,实测 raycast 清除延迟,必要时③区航点上抬从高处过(障碍高 2m——不可行则保持垂直穿越策略) |
| 低位环④与 virtual_ground 冲突 | 拒绝下到环高度 | 航段级 ground/ceil 切换(§4.2,唯一新开发项) |
| 现场 5 分钟备赛超时自动开始计时 | 手忙脚乱 | 一键启动脚本 + 演练:从上电到可起飞 <3min |
