# 仿真运行手册 — 里程碑1（真值定位）

> 依据：`空中挑战赛_Unity仿真器_ROS2_PX4全栈接入指南_路线B.md`（仓库根目录）
> 定位：用官方 Unity 当场景 + 点云渲染器，把实飞同一套 stack（EGO v1 → adapter → offboard → PX4 SITL，经 uXRCE-DDS）原样闭环跑，验 planner / 避障 / mission 调度 / yaw 行为。
> 里程碑1 = 真值当定位，暂不上 FAST-LIO，绕开"仿真无 IMU + Livox g 单位"两个坑，是本方案主战场。

---

## 0. 先决认知（别踩）

- **跑在 x86 + NVIDIA GPU 宿主机上，不是 Orange Pi 5**。Unity 渲染要 GPU；工作区要在宿主机重新 `colcon build`（aarch64 产物不能搬 x86）。
- **障碍无物理**：无人机在 Gazebo 空世界飞，障碍只在 Unity（点云）。撞不撞看轨迹有没有穿点云，不由物理引擎判（红线2）。
- **map 是 NWU 不是 ENU**：`(x,y,z)_NED=(x_map,-y_map,-z_map)`。pose_relay 真值→map 用 `(x,-y,-z)`，**且必须 rviz 现场标定符号**（红线3）。
- **只用真值喂 Unity**，绝不能喂 FAST-LIO 估计值，否则渲染→点云→估计→渲染成闭环发散（红线1）。里程碑1 两条线都用真值。

### ★ 头号坑：px4_msgs 与 SITL 固件同源

`offboard_node.cpp` 订带版本后缀的 `/fmu/out/vehicle_local_position_v1`、`vehicle_status_v3`、`failsafe_flags`、`vehicle_land_detected`（≈PX4 v1.15+）。SITL 固件与 `src/px4_msgs` 对不上 → uXRCE 话题名/哈希错 → `px4_offboard` 永卡 INIT（`pos_valid_` 永假）。

起 SITL 后先核对：
```bash
ros2 topic list | grep fmu    # 四个话题名要与代码一字不差
```

## 2. Docker 容器：Unity + ros_tcp_endpoint（ROS1 侧）

只编 `ros_tcp_endpoint`，`diff_planner / mars_drone_sim / cascadePID` 全部 CATKIN_IGNORE 不编。

```bash
xhost +local:root        # 宿主机放行 X11
docker run -it --name aac_sim --gpus all --network host \
  --env DISPLAY=$DISPLAY --env QT_X11_NO_MITSHM=1 \
  --env NVIDIA_DRIVER_CAPABILITIES=all \
  --volume /tmp/.X11-unix:/tmp/.X11-unix:rw \
  --volume $HOME/Aerial_Autonomy_Challenge:/root/ws \
  osrf/ros:noetic-desktop-full bash
# 容器内:
cd /root/ws
for p in Utils diff_planner uav_simulator user_command \
         Simulator-bridge/unity_robotics_demo Simulator-bridge/unity_robotics_demo_msgs; do
  touch src/$p/CATKIN_IGNORE
done
source /opt/ros/noetic/setup.bash && catkin_make -j1 && source devel/setup.bash
```

验证：`rostopic list` 能看到 `/drone_0_pcl_render_node/cloud`。

## 3. 启动顺序 Checklist（里程碑1）

按序开，每步确认再下一步：

```
① Docker: roscore
② Docker: source devel/setup.bash && roslaunch ros_tcp_endpoint endpoint.launch tcp_ip:=127.0.0.1 tcp_port:=10000
③ Docker: ./sh_files/run_unity.sh   (或直接跑 .x86_64)
④ 宿主机: ros1_bridge   —— 同时 source noetic+humble, ROS_MASTER_URI 指容器 master:
        source /opt/ros/noetic/setup.bash
        export ROS_MASTER_URI=http://127.0.0.1:11311
        source /opt/ros/humble/setup.bash
        ros2 run ros1_bridge dynamic_bridge --bridge-all-topics
        验证: ros2 topic hz /drone_0_pcl_render_node/cloud   (点云在动)
⑤ 宿主机: MicroXRCEAgent udp4 -p 8888
⑥ 宿主机: cd ~/PX4-Autopilot && make px4_sitl gazebo-classic
        验证: ros2 topic list | grep fmu   (核对四个话题名)
⑦ 宿主机: python3 <install>/share/top_launch_pkg/launch/pose_relay.py --drone_model iris
        —— Unity 里无人机被摆到起飞点、点云刷新; ★rviz 标定坐标符号★
⑧ 宿主机: ros2 launch top_launch_pkg ego_sim_flight.launch.py enable_arm:=false
        —— dry-run 看状态机(INIT→WARMUP→...→FLY)与轨迹, 不解锁
⑨ RViz 2D Goal Pose 发目标 → 看 Unity 里避障飞行 → 再 enable_arm:=true 真闭环
```

第 ⑦⑧ 步已由本仓库代码就绪：`ego_sim_flight.launch.py` 自动去传感层、relay Unity 点云、起 pose_relay、共用实飞同一套 planner 参数与 offboard（含改动 I 的 yaw、A/B 动态参数）。

### sim launch 常用参数

```bash
ros2 launch top_launch_pkg ego_sim_flight.launch.py \
  enable_arm:=false \                 # 先 dry-run
  profile:=standard \                 # standard|conservative|fast
  mission_file:=<...>/mission.yaml \  # 非空则自动跑任务序列(替代 RViz 手点)
  sim_cloud_topic:=/drone_0_pcl_render_node/cloud \  # Unity 点云话题
  drone_model:=iris                   # SITL gazebo 机型名(x500 等按需改)
```

## 4. 上机前要填的两处标定（代码留了占位）

1. **pose_relay 坐标符号**（`pose_relay.py` 现为恒等占位）：目标 `(x,-y,-z)`，但**必须 rviz 现场标定**——让无人机沿 Gazebo +x 平移，确认 EGO map 里朝对应方向动、Unity 画面一致再固化。错一个符号轨迹镜像翻转。
2. **得分环/goal 话题名**：Unity UI "Target the ring" 发的话题 README 未标（疑 `PoseArray`）。容器里 `rostopic list` 点一次看多出什么。里程碑1 也可先用 RViz 手点 goal，不接环序列。
3. （小）确认 Unity 点云 `frame_id` 为 `camera_init`，否则 relay 前加改 frame 小节点（`ego_sim_flight.launch.py` 已留注释）。

## 5. 覆盖范围（能验 / 不能验）

**能验（主战场）**：goal 调度 → EGO 规划 → adapter → offboard → PX4 全闭环、避障、0.7m 窄缝连通性、引导航点穿环几何、区域③移动障碍残影清除、mission_executor（fly-through/航段参数/失败兜底/精降链路）、改动 I 的 yaw（起飞无 snap / 转角单调）、A/B 动态参数。

**不能验（别外推，见改动方案 §四）**：FAST-LIO 外参/振动/漂移、220mm 动力学与风扰姿态余量、碰撞/精降精度、Orange Pi 5 真实算力。这些只能真机。

## 6. 常见坑速查

| 现象 | 原因 | 处理 |
|---|---|---|
| `px4_offboard` 卡 INIT | fmu 话题名/版本对不上, `pos_valid_` 永假 | px4_msgs 与 SITL 固件同源(§1★); `ros2 topic list\|grep fmu` |
| ROS2 侧看不到 cloud | bridge 没同时 source 两套环境 / master URI 错 | §3 ④, 确认 `ROS_MASTER_URI` 指容器 master |
| Unity 里无人机不动 | 没人发 `/quad_0/lidar_slam/odom` 或没桥回 ROS1 | 查 pose_relay + bridge 的 ROS2→ROS1 方向 |
| 轨迹在 Unity 里镜像/翻转 | NED↔map 符号错(map 是 NWU) | §4.1 rviz 标定 |
| 穿过障碍没"撞" | 障碍只在 Unity, Gazebo 空世界 | 正常, 按轨迹清障判(红线2) |
